# Proxima API

FastAPI backend for Proxima.

## Current MVP

- Single-user auto-auth (`POST /auth/auto`)
- Current owner (`GET /api/me`)
- Project create/list/detail/link
- Chat sessions, runs, events, files, wiki, workflows, jobs, schedules
- SQLite tables keep some legacy user/member/invite plumbing internally, but runtime behavior is one owner

## Test

```bash
uv run python -m pytest -q
```

## Run dev server

```bash
uv run python scripts/serve.py
```

## Notes

This is a single-user cockpit. Anyone who can reach the API should be treated as the owner, so production exposure must be protected by loopback/Tailscale/Cloudflare Access or equivalent. Prompt-injected runner content must still not grant itself permission to read source, hidden files, secrets, or unrelated paths.
